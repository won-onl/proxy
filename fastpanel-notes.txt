Архитектура (важно)
===================

• Игра / целевой сайт — домен won.onl, сервер на FastVPS (FASTPANEL). Прокси шлёт Host
  как у нужного поддомена *.won.onl; голый won-onl.ru в конфиге может маппиться на
  ru.won.onl (см. public_apex_upstream_subdomain в config.php).

• Прокси (PHP: index.php, proxy.php) — на ОТДЕЛЬНОМ российском хостинге/VPS.
  В браузере игроков — зона won-onl.ru (DNS → IP именно РФ-сервера).

FastPanel на стороне won.onl (origin) — что проверить
=====================================================

Прямые запросы с РФ-хостинга к IP origin (тот адрес, что в DNS A у won.onl) НЕ идут через Cloudflare.
На сервере игры должно быть разрешено:

• Файрвол / security: если стоит ограничение «только IP Cloudflare» — добавьте
  в whitelist IP вашего РФ-VPS (откуда идёт cURL прокси).

• Cloudflare «Authenticated Origin Pulls» / строгая проверка клиентского сертификата
  к origin — прямой cURL с РФ может блокироваться; отключение или исключение для
  вашего сценария — на стороне панели/origin.

• Режим SSL CF→origin (Flexible / Full) влияет на то, слушает ли origin :80 или :443;
  в config.php уже есть upstream_use_http и retry на HTTP при ошибке TLS.

Российский хостинг (где лежит только прокси, won-onl.ru)
========================================================

Здесь НЕ обязательно FastPanel — может быть Beget, Timeweb, shared и т.д.

• Создать сайт с доменом won-onl.ru, указать document root, куда залиты index.php,
  proxy.php, config.php.

• SSL для won-onl.ru на ЭТОМ сервере (Let’s Encrypt и т.п.) — для игроков из РФ.

• Чтобы все пути шли в index.php: либо работающий .htaccess (Apache), либо для
  Nginx+PHP-FPM — try_files → /index.php (см. nginx-php-fpm-snippet.conf).

• Wildcard *.won-onl.ru: DNS A/AAAA на IP РФ-сервера + в панели алиас/поддомены,
  если панель это поддерживает.

Страница FastPanel «Why am I seeing this page?» в браузере на won-onl.ru
-----------------------------------------------------------------------

1) В адресной строке won-onl.ru, но ответ — как у FastPanel?  
   Откройте /verify.php в корне зеркала. Если НЕ видите строку «proxy_wononl_ru_ok»,
   то запрос НЕ попадает на РФ-хостинг с прокси. Частая ошибка: A-запись won-onl.ru
   указывает на IP сервера FastVPS (тот же, что won.onl). Тогда браузер сразу
   стучится на origin с Host: won-onl.ru — сайта с таким именем на FastVPS нет →
   заглушка панели. Исправление: A/AAAA для won-onl.ru → только IP РФ-хостинга.

2) Если /verify.php показывает «ok», а прокси всё равно отдаёт страницу с origin,
   на FastVPS в FASTPANEL должен быть создан сайт с именем, которое шлёт прокси
   (например ru.won.onl для корня зеркала). Host и SNI в curl = это имя;
   upstream_direct_ip в config.php должен быть задан на сервере.
